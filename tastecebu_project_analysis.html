Bugs & issues to fix
Hardcoded homepage stats Views/Home/Index.cshtml
The stats strip shows "200+ Restaurants, 500+ Dishes, 10k+ Foodies" but these are static strings — the DB only has 8 restaurants. Replace with real counts from store.Count().
DishListing.RestaurantId is orphaned Models/GastroData.cs
The RestaurantId field exists on every dish but it's never populated in the seed, never queried in SqliteDataStore, and the Restaurant detail page never shows its dishes. Either wire it up or remove it.
Rating never recalculates after reviews Services/SqliteDataStore.cs
RestaurantListing.Rating is set at seed time only. When users post reviews via AddReview(), the restaurant's rating is never updated. Add a call to recalculate the average rating after every review add/edit/delete.
Admin seed password mismatch appsettings.json SqliteDataStore.cs
appsettings.json declares ChangeMe123! but the Seed method hardcodes Admin123! — the config value is never read. Read the password from IConfiguration instead of hardcoding it.
DishListing has no Description field but one is passed Controllers/AdminController.cs
AddDish() receives a description parameter and silently folds it into Tags when tags are empty. Add a proper Description property to the model.
No ViewModels — all data via ViewData dictionary Controllers/*.cs
Every controller passes data as untyped ViewData["key"] strings. A mistyped key silently returns null and causes a runtime cast exception. Create strongly-typed ViewModels for each view.
Photo upload has no file type or size validation Controllers/AdminController.cs
SavePhotoAsync() accepts any file extension and has no size limit. Add a whitelist (jpg, png, webp) and a max size check (e.g. 5 MB) before writing to disk.
EF Core packages on initial 8.0.0 release Gastro.csproj
Version 8.0.0 has known bugs patched in later patch releases. Bump to the latest 8.0.x (currently 8.0.13).
Features to add
High priority
Search & filter
No search bar anywhere. Add keyword search for restaurants and dishes, filter by category/price/rating, and a sort control (by rating, newest, price).
Pagination
All listing pages load every record at once. Add server-side pagination (or at minimum infinite scroll) to restaurants, dishes, and events.
Map view for restaurants
A food guide for Cebu needs a map. Integrate Google Maps / Leaflet. Store lat/lng on RestaurantListing and render pins on the restaurants page.
Password reset
There's no "Forgot password" flow. Add an email-based reset token system (or at minimum an admin reset tool).
Medium priority
Stories section (currently empty)
StoriesController returns a static view with no data. Add a StoryPost model (title, content, author, photo, date) and wire it up.
Social sharing
Add OpenGraph meta tags to restaurant/dish/event detail pages so sharing on Facebook or Viber renders a proper preview card.
Image optimization
Uploaded photos are stored as-is. Add server-side resizing (e.g. with ImageSharp) to cap uploads at 800px wide and generate thumbnails for list views.
Email notifications
Add email confirmation on register and event reminder emails 24h before a registered event. Use MailKit or a transactional email service.
